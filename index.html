Artifact 1: Animal Rescue System (Java to C++)

Download Artifact 1

The Animal Rescue System is a program designed to manage a database of rescue animals, allowing users to intake new animals, display lists categorized by type or availability, and reserve animals for specific tasks. Originally developed in Java during my first year in school, the program uses an object-oriented approach to model animals and their attributes, leveraging modular design principles to ensure maintainability and scalability. This artifact was created as part of my coursework in Southern New Hampshire University’s computer science program, showcasing foundational software design principles and best practices.

I selected this artifact for inclusion in my ePortfolio because it highlights my ability to adapt and enhance existing software while demonstrating proficiency in multiple programming languages. This conversion project involved translating the Animal Rescue System from Java to C++, showcasing my versatility and understanding of platform-specific features. Key components of the artifact that showcase my skills include Object-Oriented Design: The program’s architecture, featuring classes such as RescueAnimal, Dog, and Monkey, reflects my understanding of inheritance and polymorphism. Cross-Language Adaptability: The translation process demonstrates my ability to navigate syntactic and conceptual differences between Java and C++. Performance Optimization: Leveraging C++'s memory management features allowed me to enhance the efficiency of data handling. This artifact was improved by restructuring the code to align with C++ standards, adding detailed comments for maintainability, and implementing additional error handling mechanisms to address potential runtime issues.

This enhancement primarily aligns with Course Outcome 3: designing and evaluating computing solutions that solve a problem using algorithmic principles and managing trade-offs in design choices. By reprogramming the application from Java to C++, I evaluated the trade-offs between the languages, such as Java’s garbage collection versus C++'s manual memory management. This project also aligns with Outcome 4 and 5: Demonstrating the use of innovative techniques by leveraging C++ features like direct memory access and efficient data structures; Developing a security mindset through the implementation of secure coding practices, such as input validation and exception handling.

Enhancing and modifying this artifact provided me with valuable learning experiences and insights into software development. One significant challenge was navigating the differences in memory management between Java and C++. While Java’s automatic garbage collection simplifies memory handling, C++ requires explicit allocation and deallocation, which forced careful consideration to avoid memory leaks and ensure program stability. Through this process, I deepened my understanding of language-specific features such as adapting algorithms and structures to suit C++’s strengths.

Artifact 2: Encryption Program (XOR to AES)

Download Artifact 2

The encryption program was originally created in August 2024 as part of my CS-405 Secure Coding course. The initial version of the program utilized XOR encryption, which, while simple and effective for certain applications, lacked the robust security measures necessary for modern data protection. Recognizing the need for a more secure encryption method, I enhanced the program by integrating AES encryption using the OpenSSL library. This transition not only improved the security of the application but also introduced me to the complexities of implementing industry-standard cryptographic techniques.

I included this artifact in my ePortfolio because it demonstrates my ability to apply secure coding practices and effectively integrate external libraries to enhance security. The artifact highlights my skills in algorithms and data structures, particularly in the areas of encryption, data manipulation, and file handling. The enhancement showcases my ability to work with block ciphers, manage memory safely, and handle exceptions in a structured manner. The improvements made, such as transitioning to AES encryption, implementing key generation using OpenSSL's RAND_bytes function, and refining file handling for encrypted data storage, showcase my capacity to adapt and implement best practices in security.

This enhancement aligns with two key course outcomes. First, it supports Outcome 1, which emphasizes collaboration when integrating external libraries to implement secure encryption. By utilizing OpenSSL, I ensured adherence to security best practices while also learning how to navigate and incorporate third-party libraries effectively. Second, it aligns with Outcome 2, which focuses on effectively communicating technical solutions. Through this project, I gained experience in documenting code with detailed in-line comments, explaining the benefits of AES encryption over XOR encryption, and making these concepts accessible to diverse audiences.

Reflecting on the enhancement process, I gained valuable insights into cryptographic security and the implementation of AES encryption. One of the key lessons I learned was the importance of proper key management and initialization vectors in encryption schemes. I also developed a stronger understanding of error handling when working with cryptographic functions. The biggest challenge I faced was configuring OpenSSL correctly within Visual Studio and ensuring that the necessary dependencies were properly linked. Overcoming this challenge reinforced my troubleshooting skills and my ability to resolve dependency issues in a complex development environment.

Artifact 3: Animal Rescue System (SQLite Integration)

Download Artifact 3

The Animal Rescue System is a program designed to manage a database of rescue animals, allowing users to intake new animals, display lists categorized by type or availability, and reserve animals for specific tasks. Originally developed in Java during my first year in school, the program uses an object-oriented approach to model animals and their attributes, leveraging modular design principles to ensure maintainability and scalability. This artifact was created as part of my coursework in Southern New Hampshire University’s computer science program, showcasing foundational software design principles and best practices.

I selected this artifact for my ePortfolio because it emphasizes my ability to enhance database functionality within a real-world application. The Animal Rescue System was initially limited to in-memory data storage, restricting its scalability and persistence. By integrating SQLite, I improved data management, ensuring that animal records are stored, retrieved, and managed efficiently. This enhancement showcases my proficiency in database design, structured query implementation, and secure data handling.

The primary improvement involved transitioning the application from in-memory storage to a fully functional SQLite database. I implemented structured tables for storing dog and monkey records, ensuring proper indexing for optimized retrieval. Using parameterized queries and prepared statements improved data security by preventing SQL injection vulnerabilities. The retrieval functions were refined to handle potential NULL values, ensuring that data integrity was maintained when fetching and displaying records. I also introduced a user-interactive menu that allows users to dynamically manage the database, including adding, retrieving, and reserving animals.

This enhancement aligns with Outcome 4: Demonstrate an ability to use well-founded and innovative techniques, skills, and tools in computing practices for the purpose of implementing computer solutions that deliver value and accomplish industry-specific goals. By integrating SQLite, I applied modern database management techniques to enhance the functionality of the Animal Rescue System. The implementation of efficient data storage and retrieval operations ensured that the system could handle a growing dataset while maintaining performance. Using best practices for secure data transactions reinforced industry-standard coding principles.

Enhancing the database functionality presented several challenges, particularly in ensuring data persistence and retrieval accuracy. One of the biggest hurdles was ensuring that inserted records were properly stored and retrieved, as early implementations resulted in unexpected outputs. Debugging these issues required a deep dive into SQLite’s data retrieval mechanisms, particularly handling NULL values and memory management in C++.
